1.A. Le box-model

Source Vincent De Oliveira.

1.B. La largeur

Définir un width: 100% est souvent une mauvaise idée.

1.B. La largeur

Les éléments blocks s’empilent de haut en bas. On parle de flux vertical.

1.C. La hauteur

Avec la valeur par défaut, height: auto, la hauteur d’un élément block s’adapte à son contenu.

1.D. Les marges

Attention, les marges en pourcentages sont surprennantes !

1.D. Les marges

Les marges verticales fusionnent :

  • avec celles du parent qui n’a pas de contenu textuel, border ou padding.
  • avec celles des éléments frères.

1.D. Les marges

Les marges peuvent servir à positionner horizontalement un élément block à l'intérieur d'un autre.

1.E. Le box-sizing

Un box-model qui n'est pas du tout pratique pour la vie des projets.

1.E. Le box-sizing

La solution avec box-sizing (support IE8+).

1.F. Conclusion

  • Toujours utiliser box-sizing: border-box ;
  • Les éléments block ont un flux seulement vertical ;
  • Il vaut mieux privilégier le padding au margin pour éviter les fusions ;
  • Il vaut mieux éviter les hauteurs et les marges en pourcentage.

2. Les flotants

2.A. Le flux entre éléments frères

Les éléments flotants n'ont pas d'empreinte dans le flux pour ses frères.

2.A. Le flux entre éléments frères

En créant un BFC, on peut créer deux colonnes sans chevauchement.

Source Le contexte de formatage block par Raphaël Goetter.

2.B. Le flux entre parent et enfant

Les éléments flotants n'ont pas d'empreinte dans le flux pour leur parent.

2.B. Le flux entre parent et enfant

On peut rétablir le flux à l'intérieur du parent, en créant un BFC ou à l'aide d'un pseudo-élément.

Source Le micro-clearfix de Nicolas Gallagher.

2.C. Conclusion

Les flottants sont des éléments compliqués à styler parce qu'ils n'ont pas d'enpreintes dans le flux. En échange, le support est très bon. Ils sont à privilégier pour le mise en page d'un nombre déterminé de colonnes avec des tailles connues. Par exemple pour réaliser des media objects.

3. Les tableaux

3.A. Des colonnes à largeur fluides

Les tableaux en CSS permettent de répartir l'espace entre un nombre indéterminé de colonnes et sans fixé (support IE8+).

3.B. Des colonnes à largeur fixes

Les tableaux en CSS permettent également de répartir l'espace à l'aide de poids.

3.C. Du responsif

Les colonnes des tableaux peuvent être transformées en lignes. C'est très pratique sur mobile !

3.D. De la réorganisation

Les lignes des tableaux ensuite être réordonnées.

3.D. Conclusion

Les tableaux ont quand-même quelques défauts :

  • Pas de marges sur les cellules ;
  • Pas de passage à la ligne ;
  • etc.

C'est une technique de mise en page à privilégier lorsque que l'on veut

  • Plusieurs colonnes doivent avoir une largeur fluides ;
  • Un nombre de colonnes variable.

4. Conclusion

Nous avons vu la mise en page à l'aide de flottants et de tableau CSS. Ils s'agit de deux outils puissants et difficile à maîtriser. Dans notre trousse à outils, nous utilisons également régulièrement :

  • Les inline-blocks
  • Les flexboxs
